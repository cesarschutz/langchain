Pasta 1: Fundamentos

Aprenda os conceitos básicos de LangChain e como trabalhar com modelos de linguagem

1 Hello World com ChatOpenAI

Como usar modelos OpenAI (GPT) com parâmetros de temperatura e modelo

ChatOpenAI temperature invoke() load_dotenv()

PY Conceitos Python Necessários

  • Imports: Como importar módulos e classes
  • Instanciação: Criar objetos com Class(params)
  • Métodos: Chamar funções de objetos com .method()
  • Variáveis de ambiente: Usar os.getenv() para API keys
from langchain_openai import ChatOpenAI
from dotenv import load_dotenv

# Carrega variáveis do arquivo .env
load_dotenv()

# Cria modelo com parâmetros
model = ChatOpenAI(
    model="gpt-3.5-turbo",
    temperature=0.5  # 0 = determinístico, 1 = criativo
)

# Faz chamada ao modelo
response = model.invoke("Olá, mundo!")
print(response.content)

Conceitos-Chave

  • temperature: Controla aleatoriedade (0.0 a 1.0)
  • model: Especifica versão do GPT
  • invoke(): Método principal para chamar o modelo
  • .content: Acessa o texto da resposta
Dica

Sempre configure suas API keys no arquivo .env para manter segurança. Nunca commite keys no código!

2 Modelos Genéricos com init_chat_model

Como usar modelos de diferentes provedores (Google, OpenAI, Anthropic)

init_chat_model model_provider Google Gemini Flexibilidade

PY Conceitos Python Necessários

  • Parâmetros nomeados: func(param="value")
  • Strings: Diferentes provedores como strings
  • Abstração: Mesma interface para diferentes implementações
from langchain_core.messages import init_chat_model

# Flexível para trocar entre provedores
gemini = init_chat_model(
    model="gemini-2.0-flash",
    model_provider="google_genai"
)

# Mesma interface para qualquer provedor

Conceitos-Chave

  • Portabilidade: Troque modelos sem mudar código
  • Provedores: google_genai, openai, anthropic
  • Abstração: Interface unificada para todos

3 Templates de Prompt Simples

Como criar prompts reutilizáveis com variáveis dinâmicas

PromptTemplate input_variables format() Reutilização

PY Conceitos Python Necessários

  • f-strings: {variable} em strings
  • Listas: ["item1", "item2"]
  • Métodos format: Substituição de variáveis
  • Templates: Padrões reutilizáveis
from langchain.prompts import PromptTemplate

# Cria template reutilizável
template = PromptTemplate(
    input_variables=["nome", "idade"],
    template="Olá {nome}, você tem {idade} anos. Conte uma piada sobre sua idade!"
)

# Usa com diferentes valores
prompt1 = template.format(nome="João", idade=25)
prompt2 = template.format(nome="Maria", idade=30)

Conceitos-Chave

  • Variáveis: Use {var} para placeholders
  • Reutilização: Um template, múltiplos usos
  • Manutenção: Mude template uma vez, afeta todos

Pasta 2: Chains e Processamento

Aprenda a criar pipelines de processamento complexos com LCEL

1 LCEL e Pipe Operator

Como conectar componentes em sequência usando o operador |

LCEL Pipe | Chains Sequencial

PY Conceitos Python Necessários

  • Operador |: Pipe operator para encadeamento
  • Composição: Output de A vira input de B
  • Fluxo de dados: Esquerda para direita
from langchain.prompts import PromptTemplate
from langchain_openai import ChatOpenAI

# Componentes individuais
prompt = PromptTemplate.from_template("Traduza para inglês: {texto}")
model = ChatOpenAI()

# Conecta com pipe operator
chain = prompt | model

# Executa pipeline completo
result = chain.invoke({"texto": "Olá mundo"})

Conceitos-Chave

  • LCEL: LangChain Expression Language
  • Pipe |: Conecta componentes sequencialmente
  • Chain: Pipeline de processamento
  • invoke(): Executa toda a chain
Dica

O operador | funciona como Unix pipes: prompt | model | parser = dados fluem da esquerda para direita

2 Decorator @chain

Como criar funções customizadas que funcionam em chains

@chain Decorator Dict I/O Custom

PY Conceitos Python Necessários

  • Decorators: @decorator modifica funções
  • Dicionários: Input e output como dict
  • Type hints: def func(x: dict) -> dict
from langchain_core.runnables import chain

@chain
def processar_texto(inputs: dict) -> dict:
    texto = inputs["texto"]
    # Processamento customizado
    texto_processado = texto.upper()
    return {"resultado": texto_processado}

# Usa em chains
chain_completa = prompt | model | processar_texto

Conceitos-Chave

  • @chain: Transforma função em componente LCEL
  • Dict I/O: Sempre recebe e retorna dicionário
  • Integração: Funciona com pipe operator

3 RunnableLambda

Como integrar funções simples em chains

RunnableLambda Lambda Simple I/O

PY Conceitos Python Necessários

  • Lambda: lambda x: x * 2
  • Funções simples: Input direto, output direto
  • Wrapping: Envolver funções existentes
from langchain_core.runnables import RunnableLambda

# Função simples (não dict)
def contar_palavras(texto: str) -> int:
    return len(texto.split())

# Transforma em componente LCEL
contador = RunnableLambda(contar_palavras)

# Usa em chain
chain = prompt | model | RunnableLambda(lambda x: x.content.upper())

Conceitos-Chave

  • Simplicidade: Para funções com I/O direto
  • Lambda: Funções inline rápidas
  • Flexibilidade: Qualquer função Python

Referência Rápida - Chains

LCEL Básico:
chain = prompt | model | parser
@chain decorator:
@chain
def func(dict) -> dict
RunnableLambda:
RunnableLambda(lambda x: x)
Invoke chain:

Pasta 3: Agentes e Tools

Crie agentes inteligentes que podem usar ferramentas e tomar decisões

1 Agente ReAct com Tools

Como criar agentes que pensam (Reason) e agem (Act) usando ferramentas

ReAct @tool AgentExecutor Reasoning

PY Conceitos Python Necessários

  • Decorators: @tool para criar ferramentas
  • Docstrings: Descrição para o agente entender
  • Listas: Lista de ferramentas disponíveis
from langchain.agents import create_react_agent, AgentExecutor
from langchain.tools import tool

@tool
def calculadora(expressao: str) -> str:
    """Calcula expressões matemáticas"""
    return str(eval(expressao))

@tool
def buscar_web(query: str) -> str:
    """Busca informações na web"""
    return f"Resultados para: {query}"

# Cria agente
agent = create_react_agent(llm, [calculadora, buscar_web], prompt)
executor = AgentExecutor(agent=agent, tools=[calculadora, buscar_web])

# Agente decide qual ferramenta usar
result = executor.invoke({"input": "Quanto é 25 * 4?"})

Conceitos-Chave

  • ReAct: Reasoning + Acting
  • Tools: Funções que o agente pode chamar
  • Autonomia: Agente decide quando e qual tool usar
  • Iterativo: Pode usar múltiplas tools em sequência
Atenção

Nunca use eval() em produção! Este é apenas um exemplo. Use bibliotecas seguras como numexpr.

Pasta 4: Gerenciamento de Memória

Adicione contexto e memória às suas conversas com IA

1 Armazenamento de Histórico

Como fazer o modelo lembrar de conversas anteriores

InMemoryChatMessageHistory RunnableWithMessageHistory Sessions

PY Conceitos Python Necessários

  • Dicionários: Armazenar sessões
  • Funções: get_session_history
  • Estado: Manter contexto entre chamadas
from langchain_core.chat_history import InMemoryChatMessageHistory
from langchain_core.runnables import RunnableWithMessageHistory

# Armazena histórico por sessão
session_store = {}

def get_session_history(session_id: str):
    if session_id not in session_store:
        session_store[session_id] = InMemoryChatMessageHistory()
    return session_store[session_id]

# Chain com memória
chain_with_memory = RunnableWithMessageHistory(
    chain,
    get_session_history,
    input_messages_key="input",
    history_messages_key="history"
)

# Mantém contexto
config = {"configurable": {"session_id": "user123"}}
response1 = chain_with_memory.invoke({"input": "Meu nome é João"}, config)
response2 = chain_with_memory.invoke({"input": "Qual meu nome?"}, config)
# Response2 lembrará que o nome é João!

Conceitos-Chave

  • Sessões: Histórico separado por usuário/conversa
  • Persistência: Mensagens mantidas entre chamadas
  • Contexto: Modelo "lembra" informações anteriores

2 Sliding Window

Como controlar tamanho da memória para economizar tokens

trim_messages max_tokens Cost Control 
from langchain_core.messages import trim_messages

def prepare_inputs(payload: dict) -> dict:
    raw_history = payload.get("raw_history", [])
    
    # Mantém apenas últimas mensagens
    trimmed = trim_messages(
        raw_history,
        token_counter=len,  # Função para contar tokens
        max_tokens=100,     # Limite de tokens
        strategy="last",    # Mantém últimas mensagens
        start_on="human"    # Começa do humano
    )
    
    return {"input": payload["input"], "history": trimmed}

Conceitos-Chave

  • Window: Janela deslizante de memória
  • Economia: Reduz custos limitando tokens
  • Trade-off: Esquece informações antigas

Pasta 5: Loaders e Banco de Dados Vetoriais

Construa sistemas RAG com busca semântica em documentos

1 WebBaseLoader

Como carregar conteúdo de páginas web

WebBaseLoader HTTP Scraping 
from langchain_community.document_loaders import WebBaseLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter

# Carrega página web
loader = WebBaseLoader("https://example.com")
docs = loader.load()

# Divide em chunks
splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,
    chunk_overlap=100
)
chunks = splitter.split_documents(docs)

3 Ingestão em PGVector

Como armazenar documentos em banco vetorial PostgreSQL

PGVector OpenAIEmbeddings PostgreSQL

PY Conceitos Python Necessários

  • Embeddings: Vetores numéricos de texto
  • Similaridade: Distância entre vetores
  • Indexação: Busca eficiente em vetores
from langchain_openai import OpenAIEmbeddings
from langchain_postgres import PGVector

# Cria embeddings
embeddings = OpenAIEmbeddings(model="text-embedding-3-small")

# Conecta ao banco
store = PGVector(
    embeddings=embeddings,
    collection_name="documentos",
    connection="postgresql://user:pass@localhost/db",
    use_jsonb=True
)

# Armazena documentos
store.add_documents(documents=chunks, ids=ids)

Conceitos-Chave

  • Embeddings: Representação vetorial de texto
  • 1536 dimensões: Tamanho do vetor OpenAI
  • Similaridade: Busca por significado, não palavras

4 Busca Vetorial

Como encontrar documentos similares usando busca semântica

similarity_search Score Semantic 
# Busca semântica
query = "Como configurar autenticação?"
results = store.similarity_search_with_score(query, k=3)

# Resultados ordenados por similaridade
for doc, score in results:
    print(f"Score: {score:.2f}")  # Menor = mais similar
    print(f"Conteúdo: {doc.page_content}")
    print(f"Metadados: {doc.metadata}")

Conceitos-Chave

  • Score: 0.0 = idêntico, >1.0 = diferente
  • k: Número de resultados
  • Semântica: Busca por significado

Async/Await e Concorrência

Como executar operações assíncronas em LangChain

import asyncio
from langchain_openai import ChatOpenAI

# Versão assíncrona
async def processar_multiplos():
    model = ChatOpenAI()
    
    # Executa em paralelo
    tasks = [
        model.ainvoke("Pergunta 1"),
        model.ainvoke("Pergunta 2"),
        model.ainvoke("Pergunta 3")
    ]
    
    results = await asyncio.gather(*tasks)
    return results

# Executa
asyncio.run(processar_multiplos())
Dica

Use ainvoke() para versões assíncronas. Muito mais rápido para múltiplas chamadas!

Tratamento de Erros

Como lidar com erros de API e rate limits

from tenacity import retry, stop_after_attempt, wait_exponential
import openai

@retry(
    stop=stop_after_attempt(3),
    wait=wait_exponential(multiplier=1, min=4, max=10)
)
def chamar_com_retry():
    try:
        response = model.invoke("Pergunta")
        return response
    except openai.RateLimitError:
        print("Rate limit, tentando novamente...")
        raise
    except Exception as e:
        print(f"Erro: {e}")
        raise

Melhores Práticas

Dicas essenciais para projetos profissionais com LangChain

Controle de Custos
  • Use temperature=0 para respostas consistentes e baratas
  • Implemente cache para queries repetidas
  • Use modelos menores quando possível (gpt-3.5 vs gpt-4)
  • Limite tokens com max_tokens
  • Monitore uso com OpenAI dashboard
Segurança
  • Valide inputs do usuário antes de passar ao modelo
  • Nunca use eval() com conteúdo gerado
  • Implemente rate limiting em produção
  • Use secrets manager para API keys em produção
  • Monitore logs para uso anormal
Performance
  • Use async/await para múltiplas chamadas
  • Implemente streaming para respostas longas
  • Cache embeddings frequentemente usados
  • Use batch processing quando possível
  • Otimize chunk size para seu use case
